攻克 task_4：不可见门的系统性处理

深渊房的门在像素上本就不可见（渲染器把深渊画在门之后）——这不是感知误差，而是环境事实，需要感知与行为双层应对。
感知标签对齐像素真相 — 被深渊覆盖的门标为不可见（不可见之物不该被标注），标签由此可学习。
穿房登记竞态根治 — 贴边对齐会先于推门标志触发引擎传送；计划外跳变用「网格大换 ∨ 对侧边界落点」证据 + 最后移动方向登记，里程计不再被污染。
行为层知识闭环 — 盲探针（推一下让引擎裁决门是否存在）→ 已探测记忆 → 危险覆盖方向驱动旋桥 → 强/弱 pending 分级路由 → 钥匙增加时重探（持钥匙推不可见锁门会被引擎直接放行）。

感知模型迭代线

v1 — 训练崩坏：图像 token 展开导致标签掩码错位 → 24% 格式失败、网格 67%。实体定位已 100%。
v2 — 掩码修复 + 3.5× 缩放使 28px 视觉 patch 与 16px 游戏 tile 精确对齐 + 150 变体地图 → 官方地图全指标 100%，端到端 3/5 全通。
v3 — 3000 个单次使用布局杜绝记忆 → held-out all_ok 0%→95.5%、格式失败 25%→0%；task_5 通关（1137≈上界1136）。
v3b（最终） — 深渊房全类型门补样续训 + 不可见门三连修（感知标签/穿房登记/盲探针闭环）→ 端到端 5/5 全通。
